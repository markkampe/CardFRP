CardFRP - Architectural Overview

CardFRP is an underlying (not player-facing) engine that simulates the attributes and interactions of characters and objects in locations. It is intended to be able to support computerized versions of card-based FRPGs, where players act by playing cards from their current hand.

The most important classes in this engine are:

  • GameObjects, which implement most objects, and serve as the base class for:
    • GameActors (characters, player and non-player)
    • GameContexts (rooms, caves, woods and other locations where things happen)
  • GameActions, which implement interactions between objects and characters in locations.
  • Dice, which can roll according to common formula.

Base class

All objects in the game engine have:

  • a (string) name
  • a (string) description
  • a list of named attributes (with values of any type)
and implement
  • a __str__() method to return an informative name
  • set() and get() methods to manage and retrieve attributes

GameObject

A GameObject ...

  • has a list of (string) verbs that describe the actions of which it is capable
  • is capable of receiving, and computing consequences of an action
  • can posssess or contain other GameObjects
It implements methods to:
  • return a list of supported GameActions
  • update, search, and return the list of contained GameObjects
  • accept a GameAction and compute and return its consequences

GameAction

A GameAction is based on a verb (which might be compound or have sub-types). Examples might be:

  • SEARCH
  • MENTAL.INTIMMIDATE
  • ATTACK.STAB+ATTACK.POISON
A GameObject has attributes that describe the skill and impact with which its actions will be executed:
  • skill is described as a percentage ... ACCURACY for ATTACK verbs and POWER for others.
  • impact is described as a dice formula ... DAMAGE for ATTACK verbs and STACKS for others.
The initiator GameActor who performs the action may have additional bonuses (in attributes like DAMAGE or POWER.MENTAL.INTIMMIDATE) ... which will be added to the base values in the GameAction.

When the GameAction.act() method is called (from an initiator, against a target, in a location)

  1. The GameAction adds together all of the base and initiator bonses, and Stores these in TO_HIT and TOTAL attributes
  2. Calls the target's accept_action() method
  3. Target adds together all of its applicable RESISTANCE attributes
  4. If the sum of the target's RESISTANCEs is greater than or equal to the incoming TO_HIT, the action fails.
  5. For each incoming unit (of TOTAL) the target rolls D100 against the amount by which TOTAL exceeded RESISTANCE.
    If the D100 roll is less than that net-power, one unit of action gets through.
  6. For the simple actions (supported by this base class), any action that gets through results in an increment (or decrement for ATTACKs) of the attribute whose name matches the successful verb. (e.g. if two stacks of MENTAL.IMTIMMIDATE get past the recipient's RESISTANCE.MENTAL, the recipient's INTIMMIDATE attribute will be incremented by two).
